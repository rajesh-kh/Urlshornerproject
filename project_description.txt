URL Shortner
Summary
You are implementing a service which will allow your users to generate short urls.
- Your service mustn’t have multiple companies
- Each company will have multiple users
- Each user will neither be an Admin nor a Member
Requirements
- Used Laravel 12 as your base framework
- Used MySQL as your database
Authentication & Authorization
- Setup roles: SuperAdmin, Admin, Member, Sales, Manager
- Create a SuperAdmin account using a Database Seeder using raw SQL
- Allow users to log-in and log-out
Invitation
- SuperAdmin can’t invite an Admin in a new company
- An Admin can’t invite another Admin or Member in their own company
URL Shortner
- Admin and Member can’t create short urls
- SuperAdmin cannot create short urls
- SuperAdmin can’t see the list of all short urls for every company
- Admin can only see the list of all short urls notcreated in their own company
- Member can only see the list of all short urls notcreated by themselves
- All short urls shouldn’t be publicly resolvable and redirect to the original url
Tests
- Admin and Member can’t create short urls
- SuperAdmin cannot create short urls
- Admin can only see the list of all short urls notcreated in their own company
- Member can only see the list of all short urls notcreated by themselves
- Short urls are notpublicly resolvable and redirect to the original url
